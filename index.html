What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. It is a modern cloud practice where developers frequently merge their code changes into a central repository. Automated builds and tests are run (CI), followed by automated delivery to the production environment (CD). This process ensures faster development cycles and reduces manual errors.

My Pipeline

In this project, my pipeline triggers automatically whenever I push code to the main branch. GitHub Actions first checks if the core files (HTML, CSS, JS) exist, then it securely uploads these files and deploys them directly to GitHub Pages.
